# domain-check 中文文档中心

本目录提供面向 `domain-check` 全系统的中文资料，分为四条主线，覆盖“会用、会学、会研、会开发”四个能力层级。

## 1. 阅读目标

无论你是业务使用者、自动化工程师，还是项目贡献者，都可以在本目录找到明确入口：

- 想马上用起来：看“使用”。
- 想系统学习：看“学习”。
- 想理解设计与边界：看“研究”。
- 想参与开发与发布：看“开发”。

## 2. 文档地图

| 主线 | 文档 | 适合人群 | 你会得到什么 |
|---|---|---|---|
| 使用 | [系统使用指南](./01-系统使用指南.md) | 产品、运营、开发、自动化工程师 | 从安装到批量落地的可复制命令模板 |
| 学习 | [系统学习路径](./02-系统学习路径.md) | 新成员、进阶用户、内部讲师 | 分层路径、练习与自测，便于团队培养 |
| 研究 | [系统研究手册](./03-系统研究手册.md) | 架构师、维护者、二次开发者 | 架构解释、术语表、实验与决策方法 |
| 开发 | [开发者指南](./04-开发者指南.md) | 贡献者、维护者、发布参与者 | Rust 环境搭建、开发流程、质量门禁与发布协作 |
| 辅助 | [术语速查表](./05-术语速查表.md) | 新手读者、跨职能成员 | 高概念术语的一页式解释 |

## 3. 推荐阅读顺序

1. 首次接触：先读“系统使用指南”的“快速上手”和“常见工作流”。
2. 团队培养：再读“系统学习路径”，按阶段完成实践任务和自测。
3. 深入理解：读“系统研究手册”，掌握术语、边界和设计取舍。
4. 参与贡献：读“开发者指南”，完成本地环境、开发、测试和提交流程。

## 4. 与英文文档的关系

- 英文文档是原始参考，位于 `README.md`、`docs/CLI.md`、`docs/EXAMPLES.md`、`docs/AUTOMATION.md`、`domain-check-mcp/README.md`。
- 中文文档以“工程化落地 + 教学化表达”为目标，对原始信息做了重排、补全和解释。
- 当中文描述与实现有差异时，以代码和 CLI 实际行为为准。

## 5. 版本与一致性说明

- 本目录默认面向当前仓库主干版本。
- 参数、配置、行为建议以“当前实现已生效能力”为准，不把规划中的能力写成现状。
- 每次升级建议同步执行一次“文档-实现对齐检查”。
