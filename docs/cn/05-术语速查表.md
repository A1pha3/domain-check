# domain-check 术语速查表

本表用于帮助新手快速理解文档中的高频术语，建议与“系统研究手册”配合阅读。

## 1. 协议与数据相关

| 术语 | 英文 | 通俗解释 | 常见误解 |
|---|---|---|---|
| RDAP | Registration Data Access Protocol | 结构化注册数据协议，通常返回 JSON | 误以为所有后缀都完整支持 |
| WHOIS | WHOIS | 传统文本查询协议，格式常因注册局而异 | 误以为文本一定可稳定解析 |
| 注册状态 | Registration Status | 域名是否已注册、可用或无法判断 | 把 `UNKNOWN` 当作可用 |
| 证据字段 | Evidence Field | 用于解释结论来源的数据片段 | 只看最终状态不看证据 |

## 2. 域名与范围相关

| 术语 | 英文 | 通俗解释 | 常见误解 |
|---|---|---|---|
| FQDN | Fully Qualified Domain Name | 完整域名，如 `myapp.com` | 把基础名当完整域名处理 |
| TLD | Top-Level Domain | 顶级后缀，如 `com`、`io` | 忽视不同后缀的行为差异 |
| 预设 | Preset | 预定义的一组 TLD 集合 | 误以为预设覆盖所有需求 |
| 模式生成 | Pattern Generation | 根据规则批量生成候选名 | 误以为生成结果都应去查询 |

## 3. 工程与运行相关

| 术语 | 英文 | 通俗解释 | 常见误解 |
|---|---|---|---|
| 并发 | Concurrency | 同时执行多个检查任务 | 并发越高结果一定越快 |
| 限流 | Rate Limiting | 控制请求速度，保护稳定性 | 只关注速度不关注可靠性 |
| 回退 | Fallback | 主路径失败后启用备用路径 | 把回退视为异常而非设计 |
| Bootstrap | Bootstrap | 从 IANA 拉取注册表映射信息 | 误以为禁用后行为完全相同 |

## 4. 集成与交付相关

| 术语 | 英文 | 通俗解释 | 常见误解 |
|---|---|---|---|
| MCP | Model Context Protocol | AI 客户端调用工具的统一协议 | 把 MCP 当成业务逻辑本身 |
| JSON 输出 | JSON Output | 机器可读结构化结果 | 直接把文本输出当机器接口 |
| CSV 输出 | CSV Output | 适合报表和表格的结构化结果 | 缺少字段约束导致后续难处理 |
| 非交互模式 | Non-Interactive Mode | 命令运行时不等待人工确认 | 忘记加 `--yes` 导致 CI 卡住 |

## 5. 新手实用建议

- 做判断前先看证据字段，不只看状态字样。
- 自动化默认用 `--batch --json --yes`。
- 关键流程对 `UNKNOWN` 做重试和人工复核。
- 版本升级后复查参数与配置文档。
